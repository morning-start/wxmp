# 数据流动与状态管理

## 概述

本文档详细描述 `wxmp` 项目中的数据流动和状态管理机制，包括缓存策略、数据流转过程、状态转换等核心概念。

## 状态管理

### 1. 缓存状态机

```mermaid
stateDiagram-v2
    [*] --> 检查缓存文件

    检查缓存文件 --> 缓存不存在: 文件不存在
    检查缓存文件 --> 加载TimeManager: 文件存在

    缓存不存在 --> 创建新TimeManager: 初始化 meta.begin=datetime.min, meta.end=datetime.min
    创建新TimeManager --> 匹配时间范围: 调用 match_remaining_time_range()

    加载TimeManager --> 匹配时间范围: 调用 match_remaining_time_range()

    匹配时间范围 --> 完全无重叠: meta.end <= need.begin OR need.end <= meta.begin
    匹配时间范围 --> 扩展结束日期: meta.begin < need.begin < meta.end < need.end
    匹配时间范围 --> 扩展开始日期: need.begin < meta.begin < need.end < meta.end
    匹配时间范围 --> 完全覆盖: need.begin >= meta.begin AND need.end <= meta.end

    完全无重叠 --> 更新meta: meta.begin = need.begin, meta.end = need.end
    更新meta --> 返回完整范围: 返回 need_time
    返回完整范围 --> [*]: 获取完整数据

    扩展结束日期 --> 计算剩余范围: remaining = (meta.end, need.end)
    计算剩余范围 --> 更新meta结束时间: meta.end = need.end
    更新meta结束时间 --> 返回扩展范围: 返回 remaining
    返回扩展范围 --> [*]: 获取扩展数据

    扩展开始日期 --> 计算开始范围: remaining = (need.begin, meta.begin)
    计算开始范围 --> 更新meta开始时间: meta.begin = need.begin
    更新meta开始时间 --> 返回开始范围: 返回 remaining
    返回开始范围 --> [*]: 获取扩展数据

    完全覆盖 --> [*]: 返回 None (无需获取)
```

### 2. 时间范围匹配算法

```python
def match_remaining_time_range(self, need_time: T) -> T | None:
    """
    时间范围匹配算法

    四种情况：
    1. 完全无重叠 → 返回完整请求范围，替换缓存
    2. 部分重叠（需要扩展结束） → 返回扩展部分，更新缓存结束时间
    3. 部分重叠（需要扩展开始） → 返回扩展部分，更新缓存开始时间
    4. 完全覆盖 → 返回 None，无需获取
    """
    # 情况 1: 完全无重叠 (meta.end <= need.begin OR need.end <= meta.begin)
    if self.meta.end <= need_time.begin or need_time.end <= self.meta.begin:
        self.meta.begin = need_time.begin
        self.meta.end = need_time.end
        return need_time

    # 情况 2: 需要扩展结束日期 (meta.begin < need.begin < meta.end < need.end)
    elif self.meta.begin < need_time.begin < self.meta.end < need_time.end:
        remaining_range = type(need_time)(begin=self.meta.end, end=need_time.end)
        self.meta.end = need_time.end
        return remaining_range

    # 情况 3: 需要扩展开始日期 (need.begin < meta.begin < need.end < meta.end)
    elif need_time.begin < self.meta.begin < need_time.end < self.meta.end:
        remaining_range = type(need_time)(begin=need_time.begin, end=self.meta.begin)
        self.meta.begin = need_time.begin
        return remaining_range

    # 情况 4: 完全覆盖 (need_time 在 meta 范围内)
    else:
        return None
```

**可视化示例**:

```
情况 1: 完全无重叠
meta_time: |====|
need_time:           |====|
remaining:           |====|
new_meta:            |====|

情况 2: 部分重叠（扩展结束）
meta_time: |====|
need_time:  |========|
remaining:      |====|
new_meta:  |========|

情况 3: 部分重叠（扩展开始）
meta_time:      |====|
need_time: |========|
remaining: |====|
new_meta: |========|

情况 4: 完全覆盖
meta_time: |========|
need_time:  |====|
remaining: None
new_meta: |========|
```

### 3. 下载任务状态

```mermaid
stateDiagram-v2
    [*] --> 创建任务: ArticleDownloadTask

    创建任务 --> 检查文件存在: download_article_content()

    检查文件存在 --> 文件已存在: save_path.exists()
    检查文件存在 --> 重试循环: 文件不存在

    文件已存在 --> [*]: 返回 True (跳过)

    重试循环 --> 获取内容: for attempt in range(max_retries)

    获取内容 --> 转换格式: fetch_article_content()

    转换格式 --> 检查文件大小: save_article_content()
    转换格式 --> 异常处理: 转换/保存失败

    检查文件大小 --> 保存成功: file_size >= min_file_size_kb
    检查文件大小 --> 删除文件: file_size < min_file_size_kb

    删除文件 --> [*]: 返回 False (失败)
    异常处理 --> 重试判断: attempt < max_retries - 1

    重试判断 --> 获取内容: 是 (sleep 1s)
    重试判断 --> 记录错误: 否 (最后一次)

    记录错误 --> [*]: 返回 False (失败)
    保存成功 --> [*]: 返回 True (成功)
```

## 缓存策略

### 1. Cache Aside 模式

**FakeID 缓存**:

```mermaid
sequenceDiagram
    participant App as 应用
    participant Cache as 缓存
    participant DB as 数据库/API

    App->>Cache: 读取 fakeid
    alt 缓存命中
        Cache-->>App: 返回 fakeid
    else 缓存未命中
        Cache-->>App: 未找到
        App->>DB: 搜索公众号
        DB-->>App: 返回 fakeid
        App->>Cache: 写入 fakeid
        Cache-->>App: 写入成功
        App-->>App: 返回 fakeid
    end
```

**时间范围缓存**:

```mermaid
sequenceDiagram
    participant App as 应用
    participant Cache as 缓存
    participant DB as 数据库/API

    App->>Cache: 读取时间范围
    alt 缓存命中
        Cache-->>App: 返回 meta_time
        App->>App: 计算剩余范围
        alt 需要获取
            App->>DB: 获取剩余文章
            DB-->>App: 返回文章
            App->>Cache: 更新时间范围
            Cache-->>App: 更新成功
        end
        App-->>App: 返回文章
    else 缓存未命中
        Cache-->>App: 未找到
        App->>DB: 获取完整文章
        DB-->>App: 返回文章
        App->>Cache: 写入时间范围
        Cache-->>App: 写入成功
        App-->>App: 返回文章
    end
```

### 3. 缓存一致性

**写入策略**:
- 先写入临时文件
- 原子性重命名
- 避免写入过程中读取到不完整数据

- 读取前检查文件是否存在

- 异常时回退到无缓存状态

## 数据模型

### 1. TimeRange

```python
class TimeRange(BaseModel):
    """文章时间范围，用于缓存管理"""
    begin: datetime
    end: datetime = datetime.today()

    @field_serializer("begin", "end")
    def serialize_datetime(self, dt: datetime) -> str:
        """将 datetime 对象序列化为 YYYY-MM-DD 格式字符串"""
        return dt.strftime("%Y-%m-%d")

    def __contains__(self, dt: datetime) -> bool:
        """判断时间是否在范围内（包含边界）"""
        return self.begin <= dt <= self.end
```

**状态转换**:

```
初始状态: begin=2024-01-01, end=2024-12-31
         ↓
扩展结束: begin=2024-01-01, end=2025-12-31
         ↓
扩展开始: begin=2023-01-01, end=2025-12-31
```

### 2. ArticleDownloadTask

```python
class ArticleDownloadTask(NamedTuple):
    """文章下载任务"""
    url: str                    # 文章链接
    title: str                  # 文章标题
    save_dir: Path              # 保存目录
    save_file: Literal["md", "html"] = "md"  # 保存格式
    max_retries: int = 3        # 最大重试次数
    timeout: int = 30           # 请求超时时间（秒）
    date_str: str = ""          # 发布日期字符串
    account_name: str = ""      # 公众号名称
    digest: str = ""            # 文章摘要
    min_file_size_kb: int = 3   # 最小文件大小（KB）
```

**任务状态**:

```
创建 → 检查文件 → 获取内容 → 转换格式 → 检查大小 → 完成
  ↓        ↓          ↓          ↓          ↓
 跳过    失败       失败       失败       失败
```

## 并发控制

### 1. 线程池模型

```mermaid
flowchart LR
    A[任务队列] --> B[线程池]
    B --> C[Worker 1]
    B --> D[Worker 2]
    B --> E[Worker 3]
    B --> F[Worker 4]
    B --> G[Worker 5]
    C --> H[结果收集]
    D --> H
    E --> H
    F --> H
    G --> H
    H --> I[返回结果]
```

### 2. 并发参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| max_workers | 5 | 最大并发数 |
| timeout | 30 | 单个请求超时时间（秒） |
| max_retries | 3 | 最大重试次数 |

### 3. 并发安全

- 每个任务独立处理
- 文件写入使用原子操作
- 避免共享状态

## 错误处理

### 1. 错误传播

```mermaid
flowchart LR
    A[API 层] -->|抛出异常| B[Spider 层]
    B -->|捕获异常| C[记录日志]
    C -->|继续处理| D[其他任务]
    D -->|返回结果| E[用户]
```

### 2. 错误恢复

- **Token 失效**: 重新获取 token
- **网络错误**: 自动重试（最多 3 次）
- **文件写入失败**: 删除临时文件，记录错误
- **数据解析失败**: 跳过该条数据，继续处理
