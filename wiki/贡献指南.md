# 贡献指南

感谢您对 `wxmp` 项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告问题

如果您发现了 bug 或有功能建议，请：

1. 检查 [Issues](https://github.com/morning-start/wxmp/issues) 确认问题是否已被报告
2. 如果未被报告，创建新的 Issue
3. 提供详细的问题描述和复现步骤

### 提交代码

1. **Fork 项目**

   访问 https://github.com/morning-start/wxmp，点击 Fork 按钮

2. **克隆到本地**

   ```bash
   git clone https://github.com/your-username/wxmp.git
   cd wxmp
   ```

3. **创建分支**

   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **进行修改**

   - 遵循项目的代码风格
   - 添加必要的测试
   - 更新相关文档

5. **提交更改**

   ```bash
   git add .
   git commit -m "feat: 添加新功能描述"
   ```

   **提交信息格式**：

   - `feat:` 新功能
   - `fix:` 修复 bug
   - `docs:` 文档更新
   - `style:` 代码格式调整
   - `refactor:` 代码重构
   - `test:` 测试相关
   - `chore:` 构建/工具相关

6. **推送到 Fork**

   ```bash
   git push origin feature/your-feature-name
   ```

7. **创建 Pull Request**

   - 访问 https://github.com/morning-start/wxmp
   - 点击 "New Pull Request"
   - 填写 PR 描述，说明修改内容和原因

## 开发环境设置

### 1. 克隆项目

```bash
git clone https://github.com/morning-start/wxmp.git
cd wxmp
```

### 2. 创建虚拟环境

```bash
# 使用 uv
uv venv
source .venv/bin/activate  # Linux/Mac
.venv\Scripts\activate     # Windows

# 或使用 venv
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
.venv\Scripts\activate     # Windows
```

### 3. 安装依赖

```bash
# 安装项目依赖
uv sync

# 或使用 pip
pip install -e .
```

### 4. 安装开发依赖

```bash
# 使用 uv
uv sync --group dev

# 或使用 pip
pip install ipykernel
```

## 测试

### 运行测试

```bash
# 运行所有测试
pytest

# 运行特定测试文件
pytest test/test_time_range_spider.py

# 运行特定测试用例
pytest test/test_time_range_spider.py::TestMatchRemainingTimeRange::test_case1_no_overlap_meta_before_need

# 显示详细输出
pytest -v

# 显示覆盖率
pytest --cov=wxmp --cov-report=html
```

### 编写测试

- 使用 pytest 框架
- 测试文件命名：`test_*.py`
- 测试类命名：`Test*`
- 测试方法命名：`test_*`

**示例**：

```python
import pytest
from wxmp.spider.time_range_spider import TimeRange, match_remaining_time_range

class TestMatchRemainingTimeRange:
    def test_case1_no_overlap(self):
        meta_time = TimeRange(begin=datetime(2024, 1, 1), end=datetime(2024, 1, 31))
        need_time = TimeRange(begin=datetime(2024, 2, 1), end=datetime(2024, 2, 28))

        remaining_range, new_meta = match_remaining_time_range(meta_time, need_time)

        assert remaining_range.begin == datetime(2024, 2, 1)
        assert remaining_range.end == datetime(2024, 2, 28)
```

## 发布流程

### 版本号

遵循语义化版本（Semantic Versioning）：

- `MAJOR.MINOR.PATCH`
- `MAJOR`: 不兼容的 API 变更
- `MINOR`: 向下兼容的功能性新增
- `PATCH`: 向下兼容的问题修正

### 发布步骤

1. 更新版本号（`pyproject.toml`）
2. 更新 `CHANGELOG.md`
3. 创建 Git tag（格式：`v*.*.*`，如 `v1.5.0`）
4. 推送到 GitHub
5. GitHub Actions 自动发布到 PyPI
