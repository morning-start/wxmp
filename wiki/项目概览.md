# 项目概览

## 项目简介

`wxmp` 是一个微信公众号 API 工具库，提供公众号搜索、文章列表获取、文章内容下载等功能。项目采用分层架构设计，支持缓存优化和并发下载。

**项目地址**: https://github.com/morning-start/wxmp

## 核心特性

- **API 封装**: 完整封装微信公众平台后台 API
- **缓存优化**: 支持时间范围缓存，避免重复获取
- **并发下载**: 支持多线程并发下载文章内容
- **格式转换**: 自动将 HTML 文章转换为 Markdown 格式
- **灵活扩展**: 用户可直接使用已有爬虫或基于 API 构建自定义爬虫

## 分层架构

### 1. API 层 (`api/`)

提供基础的 API 调用能力，封装微信公众平台的后台接口。

**核心类**:
- `WxMPAPI`: 主 API 类，提供所有基础 API 调用
- `TokenResponse`: Token 响应模型
- `SearchBizResponse`: 公众号搜索响应
- `ListExResponse`: 文章列表响应

**主要方法**:
- `_fetch_token()`: 获取访问 token（初始化时自动调用）
- `fetch_fakeid()`: 搜索公众号
- `fetch_article_list()`: 获取文章列表
- `fetch_article_content()`: 获取文章内容
- `fetch_multi_article_content()`: 异步并发获取多篇文章内容

### 2. Spider 层 (`spider/`)

基于 API 层构建的业务逻辑层，提供高级功能。

**核心类**:
- `TimeRangeSpider`: 时间范围爬虫，支持缓存优化
- `TimeRange`: 时间范围模型
- `ArticleDownloadTask`: 文章下载任务

**主要功能**:
- 公众号 fakeid 缓存管理
- 时间范围缓存优化
- 文章列表获取和过滤
- 并发下载文章内容

### 3. Tools 层 (`tools/`)

提供通用工具函数。

**主要功能**:
- 文件名清理
- JSON/HTML/Markdown 文件读写
- HTML 转 Markdown
- YAML front matter 生成
- 时间范围缓存管理（TimeManager）

## 数据流动

```mermaid
sequenceDiagram
    participant User as 用户
    participant Spider as Spider层
    participant API as API层
    participant WxMP as 微信公众平台
    participant Cache as 缓存文件
    participant Tools as Tools层
    participant TimeMgr as TimeManager

    User->>Spider: 初始化爬虫(cookies)
    Spider->>API: _fetch_token()
    API->>WxMP: GET mp.weixin.qq.com
    WxMP-->>API: 返回 token
    API-->>Spider: token 获取成功

    User->>Spider: load_or_search_bizs(gzh_names)
    Spider->>Cache: 检查 fakeid 缓存文件
    alt 缓存存在且包含所需公众号
        Cache-->>Spider: 返回 fakeid 映射
    else 缓存缺失
        Spider->>API: fetch_fakeid(query)
        API->>WxMP: GET /cgi-bin/searchbiz
        WxMP-->>API: 返回搜索结果
        API-->>Spider: 返回 SearchBizResponse
        Spider->>Cache: 保存 fakeids 到缓存
    end
    Spider-->>User: 返回 bizs 映射

    User->>Spider: search_articles_content(bizs, time_range)
    loop 遍历每个公众号
        Spider->>TimeMgr: 检查时间范围缓存
        alt 完全覆盖
            TimeMgr-->>Spider: 无需获取
        else 需要获取
            Spider->>API: fetch_article_list(fakeid, ...)
            API->>WxMP: GET /cgi-bin/appmsg
            WxMP-->>API: 返回文章列表
            API-->>Spider: 返回 ListExResponse
            Spider->>TimeMgr: 更新时间范围缓存
            TimeMgr->>Cache: 保存 CSV 文件
        end
    end
    Spider-->>User: 返回合并的 DataFrame

    User->>Spider: save_all_article_content(df, ...)
    Spider->>Tools: 并发下载任务
    par 并发下载
        Tools->>API: fetch_article_content(link)
        API->>WxMP: GET 文章链接
        WxMP-->>API: 返回 HTML
        API-->>Tools: 返回 HTML 内容
        Tools->>Tools: HTML 转 Markdown
        Tools->>Cache: 保存文章文件
    end
    Spider-->>User: 下载完成统计
```

## 状态管理

### 缓存状态

```mermaid
stateDiagram-v2
    [*] --> 检查缓存文件

    检查缓存文件 --> 缓存不存在: 文件不存在
    检查缓存文件 --> 比较时间范围: 文件存在

    缓存不存在 --> [*]: 返回完整请求范围

    比较时间范围 --> 完全无重叠: 无交集
    比较时间范围 --> 部分重叠: 有交集
    比较时间范围 --> 完全覆盖: 请求在缓存内

    完全覆盖 --> [*]: 返回 None (无需获取)

    部分重叠 --> 扩展开始日期: 需要扩展开始
    部分重叠 --> 扩展结束日期: 需要扩展结束

    扩展开始日期 --> 更新缓存: 更新缓存开始时间
    扩展结束日期 --> 更新缓存: 更新缓存结束时间

    更新缓存 --> [*]: 返回剩余范围

    完全无重叠 --> 替换缓存: 完全替换缓存
    替换缓存 --> [*]: 返回完整请求范围
```

## 使用场景

### 场景 1: 直接使用 API

适合需要精细控制 API 调用的场景。

```python
from wxmp import WxMPAPI

api = WxMPAPI(cookies)
response = api.search_fakeid("Python")
```

### 场景 2: 使用已有爬虫

适合快速获取公众号文章的场景，支持缓存优化。

```python
from wxmp.spider import TimeRangeSpider

spider = TimeRangeSpider.from_cookies_file("cookies.json")
df = spider.search_articles_content(
    bizs={"公众号名": "fakeid"},
    time_range=TimeRange(begin=datetime(2024, 1, 1))
)
```

### 场景 3: 自定义爬虫

适合需要自定义业务逻辑的场景。

```python
from wxmp import WxMPAPI
from wxmp.spider import TimeRangeSpider

class CustomSpider(TimeRangeSpider):
    def custom_method(self):
        pass
```

## 扩展指南

### 1. 扩展 API 层

继承 `WxMPAPI` 类添加新的 API 方法：

```python
class ExtendedAPI(WxMPAPI):
    def custom_api_call(self, params):
        pass
```

### 2. 扩展 Spider 层

继承 `TimeRangeSpider` 类添加新的业务逻辑：

```python
class CustomSpider(TimeRangeSpider):
    def custom_search_logic(self):
        pass
```

### 3. 扩展 Tools 层

在 `tools/` 目录添加新的工具模块：

```python
# tools/custom.py
def custom_tool():
    pass
```
